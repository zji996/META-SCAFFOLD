# Handoff

仅在工作将暂停、换会话、换 agent，或用户明确要求交接时使用。

## 原则

- 自包含：读者不需要聊天记录也能继续。
- 写事实：明确 commit、工作区状态、验证结果、阻塞和下一步。
- 不重复：稳定事实链接到仓库文档；修复细节交给 git diff/log。
- 不虚构：未运行的验证、未确认的状态和未来计划不得写成完成。

## 可粘贴模板

```text
继续 <项目/目标>。先读取 <生效的项目规则>，再读取 <current/plan/ADR 中与目标直接相关的文件>。

当前状态：
- HEAD：<短 hash + message>
- 工作区：<干净，或列出必须保留的未提交文件>
- 已完成：<结果，不写过程>

验证：
- <实际运行且通过的命令>
- <既有 warning 或未运行项及原因>

下一步：
1. <第一个可执行动作>
2. <后续动作>

约束：
- <只列本目标最容易违反的 2-4 条项目约束>
```

如果 `docs/current.md` 已能完整承载这些信息，只需给出加载入口和下一步，不再复制一份长模板。
