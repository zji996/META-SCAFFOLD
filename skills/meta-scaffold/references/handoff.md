# Handoff

仅在暂停、换会话、换 agent，或用户明确要求交接时使用。

目的：读者不依赖聊天记录也能继续。写 HEAD、工作区、结果、实际验证、阻塞、下一步；稳定事实链到仓库文档；未跑的验证不得写成完成。

```text
继续 <项目/目标>。先读取 <生效的项目规则>，再读取 <current/plan/ADR 中与目标直接相关的文件>。

当前状态：
- HEAD：<短 hash + message>
- 工作区：<干净，或必须保留的未提交文件>
- 已完成：<结果，不写过程>

验证：
- <实际通过的命令>
- <未运行项及原因>

下一步：
1. <第一个可执行动作>
2. <后续动作>

约束：
- <本目标最易违反的 2-4 条>
```

若 `docs/current.md` 已完整承载以上信息，只给加载入口和下一步即可。
